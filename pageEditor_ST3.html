Technical Description

Program Directory & Feedback

The main content is structured using semantic HTML elements such as <main>, <section>, and <header> to organize content clearly and improve readability. The <main> element acts as the primary container for the core content.

The layout is created using Tailwind CSS utility classes such as flex, grid, and lg:grid-cols-12 to ensure a responsive design. Spacing and alignment are controlled using classes like px-6, py-24, and gap-12.

The Program Directory section uses overflow-x-auto together with flex and snap-x to create a horizontal scrolling card layout. Reusable styles such as ocean-card provide a consistent design with borders, transparency, and hover effects.

The Feedback Form is structured using elements like <form>, <fieldset>, <input>, and <textarea> to group and collect user data. Styling is applied using utility classes for borders, colors, and focus states to improve usability and visual consistency.

Accessibility

Accessibility was improved by using semantic elements such as <header>, <main>, <section>, and <form>, which help screen readers understand the page structure.

Form inputs are clearly labeled using <label> elements, making it easier for users to identify each field. Required fields are indicated using the required attribute to support form validation.

Images include the alt attribute to provide alternative text for users who cannot see visual content. Buttons and links are styled with clear text and icons to improve usability.

Color contrast and focus styles are maintained using Tailwind classes such as focus:ring and visible text colors, ensuring better readability and keyboard navigation.

Team Page team.html

Technical Description

The main container uses the <main> tag to define the primary content area, improving structure and readability. Content is divided into sections using <section> tags, which logically separate the hero, team grid, and statistics areas.

A responsive layout is achieved using <div> elements combined with CSS Grid classes such as grid, grid-cols-1, and md:grid-cols-2. This allows the layout to adapt across different screen sizes. Flexbox utilities like flex and justify-center are used for alignment and positioning.

Each team member is presented using a card structure with <img> for images and <h3> / <p> for text content. The relative and absolute positioning is used to create the overlay effect, where additional details appear on hover.

Accessibility

Accessibility was considered to improve usability for all users. Images include <img alt="..."> attributes, which provide descriptive alternative text for screen readers. This ensures visually impaired users can understand the content. Interactive elements such as team cards use tabindex="0", allowing keyboard navigation. This helps users who cannot use a mouse to access all content. Color contrast is maintained using combinations like text-white on dark backgrounds to ensure readability. Text sizes such as text-xl and text-sm improve clarity. Semantic tags like <section> and headings (<h1>, <h2>, <h3>) provide a logical content structure, which assists screen readers in understanding the page hierarchy. Focus styles like focus-within:ring are used to visually indicate selected elements when navigating via keyboard.

Content Page - Coral Reef Loss content_ST3.html

Technical Description

The <main> tag wraps all primary content, improving structure and SEO. Content is divided into logical sections using <section> with unique id attributes (e.g., <section id="understanding">), enabling internal navigation. hero section uses relative positioning with an <img> background and an overlay (absolute inset-0) to create a layered visual effect. Text is centered using flex and justify-center.

Images use object-cover and aspect-[4/3] to maintain proportions. Hover effects are created with group and group-hover:opacity-50 for smooth interactivity. The statistics and action sections use reusable card components styled with classes like glass-card, rounded-ocean, and shadow-2xl to maintain design consistency. The table is structured using <table>, <thead>, and <tbody> for clear data representation. A sidebar navigation is implemented using <aside> with position: sticky (sticky top-40), allowing users to easily navigate between sections using anchor links.

Accessibility

Images include descriptive <img alt="..."> attributes, allowing screen readers to interpret visual content. Semantic elements such as <section>, <aside>, <nav>, and headings (<h1>, <h2>) create a clear content hierarchy for assistive technologies. Internal navigation links (e.g., <a href="#impact">) improve usability by allowing users to quickly jump between sections. Color contrast is maintained using classes like text-white and text-light-blue on dark backgrounds to enhance readability. Interactive elements such as buttons and links include hover and focus states (hover:, transition-all), providing visual feedback for user actions. The responsive design ensures content remains readable and functional across different devices, supporting accessibility for mobile users.

Challenges and Lessons Learned

One major challenge was creating a fully responsive layout that works well across different screen sizes. Initially, elements overlapped or did not align properly on smaller devices. This was resolved by using responsive utility classes such as grid-cols-1 and lg:grid-cols-2, along with Flexbox adjustments like flex-col and lg:flex-row. Another challenge was implementing overlay and hover effects (e.g., using relative and absolute positioning). At first, the overlays did not appear correctly. This was fixed by correctly structuring parent containers with relative and controlling child elements using absolute inset-0 and z-index. Managing consistent styling across sections was also difficult. This was improved by reusing utility classes like rounded-ocean, glass-card, and consistent spacing (px-6, py-24). From these challenges, I learned the importance of planning layout structure, testing responsiveness early, and using reusable styles. It also improved my understanding of positioning, responsive design, and clean UI development.

Compliance

The developed web pages follow key principles of JANET web page publication regulations, particularly in areas such as accessibility, usability, and responsible content design. Firstly, accessibility guidelines are considered by using semantic HTML elements such as <main>, <section>, <nav>, and <aside>, which improve structure for screen readers. Images include <img alt="..."> attributes to ensure non-text content is accessible. Additionally, keyboard navigation is supported through features like tabindex="0" and visible focus styles. Secondly, the pages maintain good usability by implementing responsive design using CSS classes such as grid, flex, and breakpoint modifiers like lg:. This ensures the website functions effectively across different devices and screen sizes, aligning with JANET’s requirement for inclusive access. In terms of content and design, clear headings (<h1>, <h2>) and structured navigation improve readability and user experience. Internal linking (e.g., <a href="#section">) supports easy navigation.
