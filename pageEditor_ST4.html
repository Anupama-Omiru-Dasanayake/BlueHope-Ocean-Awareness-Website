Technical Description

Content in main are grouped through calling class inside section then button onclick="startStep()" using this button will aceess the function and triggers JavaScript logic for collecting user input step by step.progress built in html element displays completion percentage visually. Using class as profileOutput in div sections act as a container and displays user-entered data dynamically.

Global CSS used for designing colour themes and other effects like hover etc. to improve more consistency. The flexbox layout used display:flex creates responsive layouts and aligns elements neatly without floats.CSS applied for card designs and hover effects used on buttons and cards will enhance user interaction feedback.span is styled differently to act like a label.

Main logic in this user profile is when user clicks button data collected via prompt() functions step-by-step and stored in objects created,update, track and then displayed using innerHTML.Here at first single object profile stores all data and each step has own nested object. So it'll be easy to access like profile.step1.name. And then using functions like startStep() and logically procede using conditional statements using if , else-if decides which stepfunction to run. prompt() function will collect user input and store the data if user enters something using conditional comparisons calling the objects created inside the function.document.getElementById updates text dynamically using inner Text and profile displayed through innerHTML designed using updateProgress and displayProfile functions to get the logic completed.Additionally s1.name? is a ternary operator which is the shorter-version of if-else which asks if s1.name has value showing HTML and if it's empty show nothing. Also document.querySelector() will selects the first element that matches the classes.hasData acts as a boolean flag control to track the state of completed steps and make decisions later, also it'll prevent empty steps being marked completed.
