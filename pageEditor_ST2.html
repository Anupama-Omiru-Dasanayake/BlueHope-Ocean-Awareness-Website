Splash Screensplash.html

Technical Description

The splash screen implements multiple layers of CSS animations to create a dynamic underwater environment

Sun Rays Animation: The .ray elements use CSS keyframes with dynamic rotation and scaling. Each ray has individual timing using CSS custom properties (--duration) and delays, creating varied, organic movement patterns.


Bubble Animation: Bubbles rise from the bottom to the top using complex keyframes. Each bubble has unique properties (--time, --x, --scale) for varied speeds, horizontal drift, and scaling, preventing mechanical repetition.


Fish Animation: Fish swim continuously across the screen with a animation. This creates a smooth left-to-right swim cycle, with fish flipping direction at screen edges.


Fish SVG: Ellipse bodies with polygon tails, circle eyes, and line gill details

Coral SVG: Complex path elements with custom fill colors for visual variety


Skip Button Functionality: The skip button provides user control by Clearing the timer interval with clearInterval(timerInterval), Immediately redirecting to the home page, Bypassing the countdown for users who want to proceed immediately


CSS Custom Properties: --duration: Controls animation speed for rays, --time: Sets bubble ascent duration, --x: Horizontal drift distance for bubbles --scale: Size variation for bubbles, --fish-y: Vertical position offset for fish, --fish-time: Swim cycle duration

Accessibility

The skip button provides an accessible exit point for users who find animations overwhelming or want to proceed immediately. Text elements fade in with smooth animations and maintain good contrast against dark backgrounds. The content layer is positioned above animations with a high z-index, ensuring text remains readable. The layout uses clamp() for responsive typography across screen sizes.

Impact Simulatorais.html

Technical Description

Core Structure and Layout The simulator uses a container-based layout with the main interactive elements organized in a clear, hierarchical structure. The #Simulator-container serves as the primary wrapper, containing three main sections: a title header, a grid of action cards, and a feedback panel.

The action cards are arranged using CSS Grid with grid-template-columns: repeat(auto-fit, minmax(280px, 1fr)), which creates a responsive layout where cards automatically wrap and resize based on viewport width while maintaining a minimum width of 280 pixels.


Liquid Glass Design Implementation:

Backdrop-filter blur: Each card and panel uses backdrop-filter: blur(6px) combined with background: rgba(255, 255, 255, 0.10) to create a translucent, frosted-glass effect

Box shadows: box-shadow: 0 8px 32px 0 rgba(31, 38, 135, 0.37) adds depth and dimension

Border styling: Semi-transparent borders with border: 1px solid rgba(255, 255, 255, 0.18) complete the glass effect


When the user selects actions, the calculateImpact() function evaluates the total score and applies the appropriate background image using simulatorContainer.style.backgroundImage. The background transitions smoothly with transition: background-image 0.5s ease-in-out.


Event delegation: cards.forEach(card => { card.addEventListener('click', () => {...}) }) attaches listeners to each card

Class toggling: card.classList.toggle('selected') manages the visual selection state

Keyboard accessibility: Additional keypress event listeners check for the Enter key, enabling keyboard navigation


The calculateImpact() function aggregates scores by Iterating through all cards using cards.forEach(), Checking for the selected class using classList.contains('selected'), Parsing and summing the data-impact attribute values with parseInt(), Updating the count and score displays

Accessibility

The simulator includes keyboard navigation with tabindex="0" on each card and Enter key support. Visual feedback is provided through hover effects (cards lift upward) and selected states (blue borders). Impact levels use color-coded text (red, orange, green) with sufficient contrast. The responsive layout ensures usability across devices, and all interactive elements have clear text labels describing the actions.

Content Page - Overfishing content_ST1.html

Technical Description

Content is organized within a .content-container with max-width: 900px and centered margins, creating optimal readability. Four main sections cover Introduction, Main Causes, Coastal Impact, and Solutions.


A sub-navigation bar provides anchor links to each section using href="#section-id". Smooth scrolling is enabled globally with scroll-behavior: smooth, creating seamless transitions between sections.


Four images are placed throughout the content, each styled with max-height: 400px, object-fit: cover, rounded corners, and hover scaling effects. A fixed-position "Go to Top" button (bottom-right corner) links to #top on the body element, with hover effects including background color change and upward translation.


The page uses Playfair Display for headings and Google Sans for body text. Heading hierarchy: H1 at 3rem, H2 at 2rem, with body text at 1.1rem and 1.8 line height for comfortable reading.

Accessibility

The page uses proper HTML5 elements: header, main, section with descriptive IDs, and footer. Heading hierarchy follows logical order from H1 to H2, creating a clear document outline for screen readers.


Most images include alt attributes, though three have empty alt text that should be populated with descriptive content explaining their relevance to the topic.


mages scale with width: 100%, and the content container uses relative units. The "Go to Top" button remains accessible across all viewport sizes

Challenges and Lessons Learned

Challenges:

Dynamic Background Transitions: Initially, background images changed abruptly when impact levels shifted, creating a jarring user experience. The solution was adding transition: background-image 0.5s ease-in-out to the container, smoothing the visual change.

Keyboard Accessibility: Cards were initially only clickable with a mouse. Adding tabindex="0" and keypress event listeners for the Enter key resolved this, making the simulator fully keyboard accessible.


Lessons Learned:

Progressive enhancement matters. Starting with basic functionality then layering animations and accessibility features ensures the core experience works for all users.

User feedback is visual. Small touches like hover effects and color changes significantly improve perceived responsiveness.

Compliance

The Ocean Impact Simulator, Splash Screen, and Overfishing Content Page comply with Janet regulations governing web page publication through several key implementations. Accessibility is addressed through semantic HTML5 elements, keyboard navigation with tabindex and Enter key support, descriptive alt text for images, and sufficient color contrast that meets WCAG standards. Data protection is ensured as none of the pages collect or store personal user data without consent. Content accuracy is maintained by citing credible sources such as UN Food and Agriculture Organization statistics. Security is upheld by using trusted CDNs (Google Fonts, Font Awesome) and avoiding execution of untrusted code. All pages feature responsive design for inclusive access across devices, consistent navigation, and clear language. Full compliance is verified through W3C validation and accessibility testing tools.
