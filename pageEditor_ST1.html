Home page home.html

Technical Description

Template

The template includes a global navigation bar, footer section, global java script file, and a global stylesheet, These files are reused by all other team members when creating their pages to maintain a consistant theme across the website.

The navigation bar was implemented inside the <header> element with a <nav> element to organize the navigation links to the main pages of the BlueHope web page such as Home, Gallery, Impact Simulator, Feedback, Team, and Sitemap, The navigation bar has threee parts logo part - left-side, mid links part - mid, and the profile part - right-side. These parts are separated using <div> elements and layout is created using css flexbox display: flex;. The navigation links were structured using <ul> and <li> elements to create a clear organized navigation bar. header is set to positioned as fixed using css position: fixed; property to make the navigation bar visible all the time even when scrolling at the top of the page.

The footer section includes a wave animation implemented using CSS @keyframes together with the background-position-x property. A repeating customized wave images with values plus and minus for the background-position-x is used so the animation moves smoothely as a water wave and create a realistic wave effect. The footer section is also structured using css flexbox. The footer includes a small description about the BlueHope website, quick links, take action links, editor information links, and social media links. To make the structure of the footer links css flex-direction: column; to make the elements display as columns, justify-content: space-between to get space between elements verticaly and align-items: flex-start to make the elements start from the begining horizontaly are used if the flex-direction: row;(default) justify-content property works horizontaly and align-items property works verticaly.

The style.css global stylesheet was created to store shared styles sch as color variables, header and and footer styles, and shared components like buttons, cards, and intext-links. CSS variables declared using the :root selector to maintain a consistant color theme and to easily reuse colors across different CSS properties.

Global JavaScript is linked using external JavaScript file (script.js). this script is used to control the "go to top" button, when the page is not scrolled down the button remain hidden, when the use scrolls down beyond certain distance, JavaScript make the button visible. This is done by using window.addEventListener("scroll"), which listens for scroll events on the page. When the condition is met the button becomes visible by modifying its CSS opacity and z-index properties. The script also include another scroll event listener to slightly reposition the header by going bit up when scrolling. This was done by modifying css header top property.

Additionaly, template header include external links such as Google Fonts and Font Awsome icons for icons used in header and the footer.

Home

Home page include mutiple sections such as hero section with background video and two buttons, missioon statement section, why ocean matter section with showing a description when hovering over the cards, ocean facts section showing known facts about the ocean, Threats to marine life section with four main contents linking to each content page, and lastly a small preview section for the gallery page with a linked button to the gallery page. The hero section uses the <video> element with attributes such as autoplay to play the video automaticaly, attribute loop to loops the video over and over again infinitly and attribute muted to mute the video sound. Heading title and paragraphs with buttons as placed above the video using css positioning setting the video to position: absolute; and object-fit: cover; property is making sure the video fills the entire section while maintaining width and height aspect ratio.

The HTML structure of the home page was built using key structural elements such as <section> and <div> to organize the layout into clear sections. For layout design, the display: flex; property was used together with other Flexbox properties. The flex-direction property controls whether elements are arranged in rows or columns The align-items property controls the cross axis alignment of the elements, while justify-content controls the main axis alignment. values used include center, space-between, space-around, and space-evenly. The flex-wrap property was also used to allow elemnts to wrap when the available spaxe becomes smaller.

Other CSS properties such as transition, overflow, backdrop-filter, border-radius and box-shadow were also used to improve the visual appearance and create smooth hover effects when user hover over elements on the page.

Accessibility

HTML Elements such as header, nav, main, section were used to create a clear page structure that can be understood by screen readers. Images also include with alt attributes to help users who cannot see the images undertand the what is the image is about Clear navigation links and good color match between text and background were also used to improve readability of the users. Images used in sections such as Why Oceans Matter and Threats to Marine Life include descriptive alt attributes so screen readers can describe the visual content to users who cannot see the images. Interactive elements such as buttons and links use clear text labels like Learn More and Explore More to help users understand their purpose. Visual feedback is provided through hover effects and active link styles so users can easily recognize interactive elements such as buttons and cards. The page layout also uses flexible design techniques such as display: flex; and flex-wrap; so the content adjusts properly on different screen sizes and remains readable on smaller devices.

Gallery Page gallery.html

Technical Description

The gallery page was build within the template's <main> HTML tag. The gallery images are organized inside a container using <div> elements and each image item is placed inside a <div class="img"> element. The data-name attribute is used to categorize images such as marine life, coral reefs, and ocean threats so they can filtered using JavaScript listeners when user click the Specific buttons. Popup contents are hidden by setting css display: none; display property to none and positioned centered in the screen as position: fixed; display: flex; css flexbox is used to create the structure of the popup content.

When the filter button is clicked Java Script reads the selected catogory and compare it with the data-name value of each image element. The script selects elements using methods such as documnet.querySelector() and documnet.querySelectorAll() and controls the visibility of the elements by adding or removing CSS classes using classList.add() and classList.remove(). To filter the images into different catogories first script selects all filter buttons using document.querySelectorAll(".filter-buttons button") and all gallery image containers using document.querySelectorAll(".gallery-images .img"), Thesse methods return list of DOM elements which are stores in variables filterButtons and filterableImages a function named filterImages is created to control the filtering process. When a user clicks a filter button, an event listerner attached using addEventListener("click") calls this function. The event object e is used to identigy which button was clicked using e.target . The code firtst of all remove the CSS filter-active class classList.remove() from the currently active button and add it to the clicked button using classList.add() . The function then loop over all galley image containers using forEach() loop, each image container is hidden by additing a CSS class hide, which use CSS display: none property to remove the images from the view. Then check the condition using the dataset.name property with the category of the selected filter button e.target.dataset.name. if the button category and the images catogory match, of if the selected button dataset category is "all" the hide class is removed using classList.remove() making the image container visiable again.

For the popup windows, The script first selects all the popup open buttons and popup containers using DOM selction methods such as document.querySelectorAll(). These selection return a list of elemnts which are then processed using forEach() loop. Inside the loop, an event listener is checking each button using addEventListener("click"), when a user clicks a Read More button, the fucntion excutes and uses the index value provided by the forEach() loop. The index represent the position of the current button in the list, The popup elements are stored in the same order as the buttons the script code can identify the correct popup using popup[index]">. The code classList("open") is then used to add the open css class to the corresponding popup container, which changes its visibility by changing CSS properties such as display and z-index.

When Closing the opened popup similar way is used. Close buttons are selected using querySelectorAll() and processed using forEach(). When the close button is clicked, classList.remove("open") removes the CSS class from the corresponding popup container, hiding it again.

Accessibility

All gallery images include descriptive alt attributes so users who rely on screen readers can understand what each image represents. The filter buttons use clear text labels such as Marine Life, Coral Reefs, and Ocean Threats, helping users easily understand and navigate between categories. Visual different look is provided through hover effects and active button highlighting so users can clearly see which filter is currently selected. The gallery layout was implemented using flexible design techniques so images wrap and adjust properly on different screen sizes, ensuring the content remains readable and usable across different devices Small image description is provided when user hover over the images. and in the popup window user can select the font style and the background color theme based on their preference giving the users more friendly reading environment.

Content Page - Climate Change content_ST1.html

Technical Description

The content page was implemented using structured HTML elements and CSS styling to present information about Climate Change and its impact on oceans. The page's content is organized using multiple <section> elements with unique id attributes such as introduction, warming, bleaching, statistics, and sea-levels. These IDs allow internal navigation links in the sub navigation bar to scroll directly to the corresponding sections of the page.

hero section was created using a <section class="hero"> element with a background image and large text to introduce the topic of the page.CSS properties such as position: relative;, height: 100vh and display: flex; css flexbox is used to center the text and make a full screen hero. image position is set to position: absolute; to make the as the hero background. A scroll down indicator link was added using an anchor tag <a href="#introduction"> to direct the user to the main content.

The main information on the page use a card based layout where images and text are inside <div class="content"> containers together with <img> elements. CSS Flexbox was used to control the layout of these sections using properties such as display: flex;, flex-direction, align-items and justify-content so the content is aligned properly and remains vidually balanced. The main card styles are accessed from the global style sheet.

For statistical information is used mini card components created with <div class="mini-card"> elements. These cards contain percentage indicators and short descriptions. Which are styles using CSS properties such as border-radius box-shadow, ::after and transition to create interactive hover effects. Additional styling properties such as transform: scale();, :hover, :hover::after are used to highlight the elements when hovering over them. small div element is used as decorations in left-side and right-side, element's css positioned as position: absolute; to make the content page more eye catching.

Accessibility

Images used throughout the sections such as Ocean Warming, Coral Bleaching, and Rising Sea Levels include descriptive alt attributes so screen readers can describe the visual content to users who cannot see the images. Internal navigation links in the sub-navigation bar use anchor links (for example href="#introduction" and href="#warming" ) which allow users to quickly navigate to different sections of the page. Clear headings and structured content sections help users understand the topic and improve readability.

Challenges and Lessons Learned

One of the main challenges encountered during the development process was implementing the gallery popup window and its's tools. it was difficult to correctly link each gallery image with its corresponding popup window. The main issue was making sure that when a user clicked the Read More button, the correct popup window opened without affecting other gallery items. This required me to learn from external sources such as w3schools webpage and youtube and get a better understanding of JavaScript DOM selection and event handling. The problme was resolved by using querySelectorAll to select mutiple buttons and popup elements, then using forEach() loop together with the index value to match each button with the correct corresponding popup element. By adding and removing CSS classes using classList.add() and classList.remove(), the popup visibility controlled. Through this challenge, I gained a better understanding of how JavaScript querySelectorAll and forEach works and how class manipulation can be used to create dynamic user interfaces.
